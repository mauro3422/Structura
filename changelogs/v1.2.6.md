# v1.2.6 - Etiquetas y validacion de relaciones

## Resumen

Esta iteracion hace mas didactico el panel de relaciones del laboratorio de tablas: separa con claridad Maestra, Detalle, Referencia sugerida y Relacion invalida.

## Cambios

- Se agregaron etiquetas explicitas para el rol de cada tabla en una relacion.
- Se marco como invalida la relacion cuando falta tabla destino.
- Se dejo claro que en una relacion 1:N la FK puede repetirse varias veces en el detalle sin ser un error.
- Se mantuvo la advertencia para 1:1 cuando la FK no es unica o no comparte PK.
- Se sumaron tests para 1:N, 1:1 y destino faltante.

## Impacto

- Menos confusion al explicar cardinalidad y claves.
- Mejor puente entre teoria y practica para relaciones relacionales.
- Lectura mas directa para quienes estan aprendiendo SQL y modelado.

## Verificacion

- `npm run typecheck`
- `npm test`
- `npm run build`
