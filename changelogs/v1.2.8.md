# v1.2.8 - Observaciones por tabla y columna con nombres flexibles

## Resumen

Esta iteracion ajusta el laboratorio de tablas para que las observaciones sean mas didacticas y menos invasivas:

- Los nombres en minúscula o snake_case ya no se tratan como un error.
- Las observaciones resaltan la tabla o la columna exacta que necesita atencion.
- La PK, las FKs y la cardinalidad siguen validandose sin redundancia de reglas por nombre.

## Detalles

- Se agrego un resaltado visual propio para el nombre de la tabla cuando el problema pertenece a la tabla completa.
- Se mantuvo el resaltado por columna para claves foraneas sin destino, duplicados y relaciones 1:1 a revisar.
- Se suavizo la sugerencia de formato para que funcione como aviso y no como bloqueo.

## Verificacion

- `npm run typecheck`
- `npm test`
- `npm run build`
