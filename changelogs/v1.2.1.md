# v1.2.1 - Corte final de arquitectura, tablas y router

**Fecha de Lanzamiento:** 22 de Abril de 2026

## Resumen

Esta iteración cerró varios bordes estructurales que seguían activos en la base: se separó el contrato central del sistema, se particionó el CSS más pesado de tablas, se tipó el router y se limpió el arranque de la app para reducir deuda legacy y problemas de carga en desarrollo.

## Cambios Principales

- **Contrato del core dividido**: `Module.ts` quedó más liviano y los tipos base pasaron a `moduleTypes.ts`, manteniendo la misma API pública pero con una separación más clara entre datos y lógica.
- **Tables CSS particionado**: el bloque de estilos de tablas se dividió en parciales por responsabilidad (`data-table`, `laboratory`, `relationships` y `controls`), eliminando el barrel problemático que rompía Vite.
- **Router tipado**: el router dejó de vivir en `.js` y pasó a `router.ts`, con tipos explícitos para rutas y parámetros.
- **Búsqueda y tablas más modulares**: la lógica de búsqueda y el DOM de tablas se separaron en helpers propios para bajar el tamaño de los archivos principales.
- **Limpieza del bootstrap y errores visibles**: se corrigieron mensajes de error y se eliminó el legacy que seguía generando ruido en el navegador.
- **Servidor dev estabilizado**: se reinició el grafo de Vite y se validó que la app volviera a servir correctamente tras los cambios de CSS.

## Impacto

- Se redujo la confusión que generaban los archivos monolíticos restantes.
- La base quedó más preparada para sumar tests sobre contratos y renderers.
- El flujo de desarrollo dejó de depender de un barrel CSS roto.
- La app quedó más estable para seguir probando en el IAB y extender contenido sin pelearse con estructura vieja.
