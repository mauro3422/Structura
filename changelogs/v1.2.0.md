# v1.2.0 - Migración a TypeScript y limpieza de encoding

**Fecha de Lanzamiento:** 22 de Abril de 2026

## Resumen

Esta iteración consolidó la base del proyecto para que sea más robusta, clara y fácil de extender. El foco estuvo en dos frentes: mover el núcleo del sistema a TypeScript y corregir textos corruptos de encoding que estaban afectando la lectura y la experiencia visual.

## Cambios Principales

- **Migración progresiva a TypeScript**: se tiparon el núcleo de la app, páginas, módulos, plantillas compartidas, widgets y secciones de lección para reducir errores y facilitar refactors futuros.
- **Limpieza de legacy**: se eliminaron wrappers `.js` de la ruta crítica y se dejó una fuente de verdad más clara en `.ts`.
- **Normalización de contenido**: se reescribieron módulos y páginas que tenían mojibake o caracteres rotos para volver a un español limpio y legible.
- **Estandarización del contenido**: los módulos quedaron más consistentes, con textos, iconos, descripciones y lecciones alineados al mismo patrón.
- **Validación de build y tipos**: se incorporó y verificó `typecheck` junto con el build de producción para asegurar que la base siga estable.

## Impacto

- El proyecto quedó más confiable para seguir sumando módulos, widgets y secciones.
- Se redujo la confusión que generaba tener fuentes duplicadas o texto corrupto.
- La experiencia de lectura en la app mejoró de forma directa, especialmente en el navegador in-app y en lecciones con mucho contenido.
- La base ahora está más preparada para la siguiente fase: tests, validaciones y nuevas funcionalidades.
