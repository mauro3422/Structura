# v1.2.2 - Tests de contrato y smoke checks

**Fecha de Lanzamiento:** 22 de Abril de 2026

## Resumen

Esta iteración cerró la primera capa de seguridad automática del proyecto. Se agregaron tests de contrato para el core y el router, y smoke tests de render para búsqueda y tablas. También se instaló y configuró Vitest con entorno `jsdom` para que la base quede lista para seguir creciendo con menos riesgo.

## Cambios Principales

- **Runner de tests agregado**: se incorporó Vitest con scripts de `test` y `test:watch`.
- **Contratos del core cubiertos**: `Module`, `Lesson` y `LessonBuilder` quedaron validados con tests de construcción y metadata.
- **Router validado**: se agregaron tests para rutas estáticas y dinámicas con params.
- **Smoke checks de búsqueda**: se probaron tanto el render de `search-animation` como la ejecución de búsquedas lineales y binarias.
- **Smoke checks de tablas**: se validó el render de tablas de ejemplo, interactivas y del laboratorio.
- **Corrección de textos visibles**: se limpiaron cadenas rotas en búsqueda y tablas para que los tests reflejen la UI real.

## Impacto

- Ahora hay una red básica para detectar regresiones en los contratos más importantes del sistema.
- Se redujo el riesgo de romper router, builders o renderers al seguir refactorizando.
- La base queda mejor preparada para sumar más tests sobre secciones, widgets y módulos específicos sin arrancar de cero.
